# The Monday Morning Workflow
### How to actually use this thing every week

---

You built the harness. You have the server. Now what?

This is the routine. Once it clicks, the whole system starts paying for itself.

---

## The 10-Minute Monday Check-In

Every Monday morning, open Claude Code, connect to your server, and run this one prompt:

```
Good morning. Read my CLAUDE.md and give me a quick rundown:
- Any lease renewals expiring in the next 60 days?
- Any late rent I should be following up on?
- Any open maintenance requests older than 7 days?

Pull from whatever tools we've built. If we haven't built a tracker for
something yet, just flag it and move on.
```

Claude reads your harness, checks your built tools, and gives you a prioritized list.
You deal with the list. That's the week.

---

## The Weekly Pattern

| Day | What you do with the harness |
|---|---|
| **Monday** | Morning check-in — what needs attention this week |
| **Wednesday** | Follow up on anything that came out of Monday |
| **Friday** | Owner reports if it's end of month — run the generator |
| **Any day** | Something comes up → ask Jarvis to handle it |

You don't need to be in Claude Code every day. You use it when it saves you time.

## How to Ask for Things

**The wrong way:** "Help me with rent."

**The right way:** "I have 3 tenants who are 5+ days late on rent. Tenant names are
Smith (Unit 4A), Jones (Unit 12B), and Rivera (Unit 7C). Draft a late rent notice
for each one, professional tone, include a $50 late fee per our lease terms."

The more specific you are, the better the output. Think of it like briefing an
assistant — they need context to do the job right.

---

## The Three Prompts You'll Use Most

### 1. Quick question
```
[Ask your question]. Keep it short — just tell me what I need to know.
```

### 2. Draft something
```
Draft a [notice / email / report] for [situation]. Here's the context: [details].
Give me a draft I can review and edit before it goes out.
```

### 3. Build something new
```
I want to build a tool that [does what]. Here's the problem I'm trying to solve:
[describe]. Give me a plain-English plan before you write any code.
```

---

## When to Build vs When to Just Ask

**Just ask** when:
- It's a one-time thing (one tenant, one situation)
- You need a draft you'll edit yourself
- You're figuring something out

**Build a tool** when:
- You do the same thing every week
- It involves a list of tenants or properties
- You're spending more than 30 minutes a week on it

If you're asking the same question more than twice, build the tool.

## Staying Sharp

**Once a month:** Run this prompt:
```
Look at my CLAUDE.md. Is there anything outdated — tools I've listed that we
haven't connected yet, or things I'm still doing manually that we could automate?
Give me 2-3 suggestions based on what I actually told you about my business.
```

**When something new happens in your business:** Update your CLAUDE.md.
New software, new vendor, new property — the more accurate it is, the better
every session gets.

---

## The Golden Rule

**The harness gets better the more you use it.** Every time you update CLAUDE.md,
every tool you build, every prompt that works well — you're training the system
to know your business. A PM who uses this every week for six months has a
completely different experience than one who built it and forgot about it.

Use it for the small stuff. That's where the hours add up.
